# Baspana Radar

Независимый советник по покупке квартир в Астане: подбор под приоритеты
покупателя, прозрачный скоринг цены и карточки due diligence по ЖК с
градацией достоверности (подтверждено / сообщается / слух / нет данных)
и ссылками на источники.

**Статус: T1 — деплой-скелет на dummy-данных.** Данные — реальный срез
krisha.kz от 2026-07-12 по ЖК «Времена года. Весна» + demo-заглушки по
5 соседним ЖК. Скрейпер, Supabase и LLM-генерация карточек — следующие этапы.

## Архитектура

```
batch pipeline (Python, офлайн)          runtime (этот репозиторий)
scrape → normalize → analytics  →  Supabase  →  Next.js на Vercel
        → card generator (LLM)                   форма → скоринг → карточки ЖК
```

Сейчас слой данных (`lib/data.js`) читает статические JSON из `data/`.
Интерфейс функций (`getListings`, `getComplexStats`, `getCard`) не изменится
при переходе на Supabase — меняется только их внутренность.

- `lib/scoring.js` — открытая формула балла: выгода (отклонение ₸/м² от
  медианы ЖК) + соответствие приоритетам − штраф за риск ЖК, взвешенные по
  цели покупки и толерантности к риску.
- `data/cards.json` — карточки due diligence. Правила системы: утверждение
  без источника не может быть выше уровня «слух»; пустой корпус → «нет
  данных», никогда не генерируем из воздуха; каждая карточка несёт дату
  генерации.
- ЖК с < 8 объявлениями помечаются «мало данных» — z-оценки по n=3 не
  показываем как истину.

## Запуск локально

```bash
npm install
npm run dev        # http://localhost:3000
```

## Деплой на Vercel (2 минуты)

Вариант A — через CLI:
```bash
npm i -g vercel
vercel             # ответить на 2-3 вопроса, получить URL
vercel --prod
```

Вариант B — через GitHub: запушить репозиторий → vercel.com → Add New
Project → Import → Deploy.

Env-переменные (Vercel → Project → Settings → Environment Variables):
```
NEXT_PUBLIC_SUPABASE_URL=https://xxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...   # anon key, НЕ service_role
```
Без них приложение работает в demo-режиме на JSON-срезе из `data/` —
деплоить можно до запуска пайплайна. С ними — читает Supabase.

## Дорожная карта

- **T2** — скрейпер krisha (search-страницы: ~1900 запросов на все ~38k
  объявлений Астаны), загрузка в Supabase, PII вырезается на входе
- **T3** — аналитика: hedonic-модель ₸/м², флаги аномалий, детектор
  расхождений (заявленный срок сдачи vs фактический год постройки)
- **T4** — генератор карточек: Layer 1 (только из собственных данных) для
  всех ЖК, Layer 2 (внешние источники + LLM, free tier, батчем) для топ-50
  и по запросу с кэшированием
- **T5** — таблица инфраструктурных событий (ЛРТ, школы, ТРЦ) с
  distance-decay поправками и сценарии по макро (ипотечные программы, ставка)
